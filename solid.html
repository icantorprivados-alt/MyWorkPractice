Introducción a SOLID

SOLID es un acrónimo que representa cinco principios fundamentales del diseño orientado a objetos. Su objetivo es ayudar a los desarrolladores a crear software que sea fácil de mantener, entender y ampliar con el tiempo.

Los 5 Principios Fundamentales

S - Single Responsibility (Responsabilidad Única)

Una clase debe tener una sola razón para existir y una sola tarea. Esto evita que el código se vuelva difícil de manejar.

O - Open/Closed (Abierto/Cerrado)

El software debe estar abierto para extenderse (añadir funciones) pero cerrado para modificarse (no cambiar el código que ya funciona).

L - Liskov Substitution (Sustitución de Liskov)

Las clases hijas deben poder sustituir a las clases padres sin que el programa falle.

I - Interface Segregation (Segregación de Interfaz)

Es mejor tener muchas interfaces pequeñas y específicas que una sola interfaz grande que obligue a implementar cosas innecesarias.

D - Dependency Inversion (Inversión de Dependencia)

Debemos depender de abstracciones (interfaces) y no de clases concretas para reducir el acoplamiento.
